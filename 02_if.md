## Будем говорить на одном языке

**Компилятор** - это программа, которая переводит код с языка программирования в машинный код, понятный процессору.

**Интерпретатор** - это программа, которая читает и выполняет код. Например Python или JavaScript.

**Среда разработки** - это программа, содержащая инструменты для написания и запуска кода. Например IDLE, VSCode или PyCharm.

**Консоль / Терминал** - текстовое окно, которое позволяет программисту взаимодействовать с компьютером. Например консоль IDLE - это текстовое окно, которое позволяет нам взаимодействовать с программой, которую мы написали.

## Комментарии

В коде можно писать заметки для себя и для тех, кто будет этот код читать. Комментарии упростят чтение кода для человека, а *интерпретатор* их просто проигнорирует. Для чего нужны комментарии?

* в длинном коде комментарии помогут найти, где начинается новый смысловой кусок кода;
* комментарии помогут вспомнить, что в каком месте программы происходит, когда вы вернетесь к своему коду после перерыва;
* если ваше решение задачи не "одноэтажно", я буду очень рада вашим комментариям в коде: они помогут мне легче понимать, что и зачем вы делаете.

### Как создать комментарий?

Просто напишите ваш комментарий на отдельной строке, начав эту строку с решетки #.

```python
# Это комментарий, а дальше код

'Это код. Он состоит из просто строковых данных, но все равно это код. Питон его читает и с ним работает.'

# А это опять комментарий. Питон его видит, но "пропускает мимо ушей".
```

## Логический тип данных

**Логические** или **булевы** данные могут принимать только два значения. Эти значения могут обозначаться по-разному, но смысл один: булевы данные либо истинны, либо ложны. Обозначения могут быть ИСТИНА и ЛОЖЬ, 0 и 1, Да и Нет или другие. В Python булевы данные имеют значения **True** и **False** (заглавные и строчные символы важны).

## Условный оператор `if`

`if` - это логический оператор, которому обязательно нужен свой синтаксис.

**Синтаксис в программировании** - это набор правил, определяющий, как должны быть расположены символы, операторы и команды, чтобы программа была корректной. Программа с синтаксическими ошибками иногда вообще не запускается (тогда мы видим ошибку `Invalid syntax`), а иногда запускается, но работает не так, как мы от нее ожидаем.

`if` используется для проверки условий. **После `if` пишется логическое выражение, затем ставится двоеточие, а со следующей строки с отступом пишется код, который должен быть выполнен в случае, если условие истинно.** Этот код может состоять из любого количества строк. Для того, чтобы интерпретатор понимал, что новые строки относятся к тому же блоку кода, нужно соблюдать отступы.

Код, написанный с отступом, называют *вложенным*.

```python
a = 5
if a > 0:
  print('Положительное')

>>> Положительное

a = -3
if a > 0:
  print('Положительное')

>>> 

a = 5
if a > 0:
  print('Положительное')
  print('Привет')
  print('Какой денек!')

>>> Положительное
    Привет
    Какой денек!
```

Давайте посмотрим внимательнее, как выполняется условное выражение. В коде выше мы сперва присваиваем переменной `a` какое-то значение. Затем идет `if` и логическое выражение. Логическое выражение может иметь два значения: оно либо истинно, либо ложно.
* Если выражение истинно, Python "заходит внутрь" if-а, *на следующий уровень вложенности* и выполняет весь вложенный код строчка за строчкой. 
* Если выражение ложно, Python полностью пропускает весь вложенный код и даже не читает его, как будто этот код закомментирован.

**Обратите внимание** на то, как в Python пишутся операторы сравнения:

* `>` - больше
* `>=` - больше или равно
* `<` - меньше
* `<=` - меньше или равно
* `==` - равно
* `!=` - не равно

**Не путайте** оператор сравнения `==` с оператором присвоения значения переменной `=`. Они не взаимозаменяемы и друг за друга не работают. Если оператор присвоения значения переменной использовать в логическом выражении, получим ошибку синтаксиса.

### Оператор `else`

Если число окажется нечетным, программа не станет выполнять никаких действий. Но что, если нам нужно выполнить какое-то действие в случае, если уловие оказалось ложным (не выполнилось)? Для таких случаев используется оператор `else`. `else` может идти **только после `if` и не может существовать без него. `else` пишется строго на том же уровне, что `if`, к которому он относится. Так же, как для `if`, обязательно двоеточие, а код, выполняемый в этой *ветке* программы, пишется с новой строки и с отступом**. Никакое условие после `else` не пишется, потому что этот оператор обозначает любые исходы кроме того, который описан в условии `if`.

```python
a = 5
if a > 0:
  print('Положительное')
else:
  print('Отрицательное')

>>> Положительное

a = -3
if a > 0:
  print('Положительное')
else:
  print('Отрицательное')

>>> Отрицательное
```

### Оператор `elif`

Но и это еще не все сценарии проверки условий. Бывает ведь и так, что если одно условие не выполнено, нужно проверить еще что-то. Для этого используем оператор `elif`. Это склейка из слов *else* и *if*. Синтаксис аналогичен: **после `elif` пишется условие, которое нужно проверить, затем двоеточие и вложенный код. `elif` может стоять только между `if` и `else`**.

**В одном блоке проверки условий может быть только один `if` и только один `else`. А вот `elif` между ними может встречаться сколько угодно раз.**

```python
a = 5
if a > 0:
  print('Положительное')
elif a == 0:
  print('Нуль')
else:
  print('Отрицательное')

>>> Положительное

a = -3
if a > 0:
  print('Положительное')
elif a == 0:
  print('Нуль')
else:
  print('Отрицательное')

>>> Отрицательное

a = 0
if a > 0:
  print('Положительное')
elif a == 0:
  print('Нуль')
else:
  print('Отрицательное')

>>> Нуль
```

Посмотрим, как выполняется код:
* если условие в if истинно, выполняется код, вложенный в if, а весь дальнейший код условного выражения, включая elif, else и вложенный в них код, игнорируется;
* если условие в if ложно, вложенный в него код игнорируется, и Python переходит к проверке условия в elif. Если оно оказалось истинным, "заходит" во вложенный в elif код и выполняет его. else и вложенный в него код интерпретатор полностью игнорирует.
* если и условие в elif тоже оказалось ложным, Python переходит сразу в код, вложенный в else (ведь в else не надо ничего проверять) и выполняет его.

После того, как условное выражение обработано, интерпретатор переходит к выполнению дальнейшего кода, если он есть.

#### Задание 2.

Допишем программу из задания 1. Теперь, если введенное число нечетное, мы напишем, что оно нечетное. А если число дробное, напишем, что оно дробное. Обратите внимание, что для выполнения такой задачи нам придется использовать тип данных не int(), как в предыдущей задаче, а float(), который позволит нам не потерять дробную часть.

```python
  print('дробное')
```
